Tier 1 — The Framework

Agentics / ag3ntics

A reusable framework for distributing project work across multiple AI instances, each with a dedicated role and isolated context boundary. Product-agnostic. Human-orchestrated.

Human = Direction

Meaning, judgment, intent, final approval. The orchestrator routes work, defines domains, makes calls the AI can't.

AI = Execution

Each AI instance has a bounded role, a dedicated context window, and a training trajectory that keeps it focused.

System = Flow

GitHub repos, project boards, and wikis are the coordination fabric. Handoffs between agents happen through commits, issues, and docs.

4 Coordination Layer GitHub as the handoff fabric between agents

Git Repos

Code, configs, docs. Each agent commits to its assigned repos. Branches isolate work. PRs are the handoff mechanism.

GitHub Issues

Work items, bug reports, RFIs. Agents pick up issues, work them, close them. Justin creates and prioritizes.

Project Boards

3 boards (Robo Stack, Content Ops, YKS Master). Kanban view of all work across agents.

Wikis

Architecture docs, sprint plans, agent prompts, retrospectives. The institutional memory across sessions.

CI/CD Pipelines

GitHub Actions for deploy + security scanning. Live for the-markdown (HF03). Robo-stack has 8 workflows.

Agent Handoff Protocol

No formal protocol for how one agent's output becomes another's input. Currently ad-hoc via Justin bridging context.

Context Packages

No standardized way to bootstrap a new agent session with the right context. Boot prompts exist in wiki but are manual.

Cross-Agent Traceability

No way to track which agent produced what, when. No audit trail of decisions across sessions.

Key correction from v1: The Agentics model is not a software layer inside The Markdown. It is the project execution framework — the way Justin distributes work across multiple AI instances with separate context boundaries. Each "agent" in the Agentics doc maps to an actual AI account/session, not a microservice. GitHub is the coordination fabric. Justin is the orchestrator who bridges context between isolated agents.

Implication: The biggest gap category (FW-01 through FW-06) was invisible in v1. The framework that holds everything together — agent handoffs, context bootstrapping, cross-agent traceability, role registry — has no implementation at all. It runs entirely through Justin's manual effort.
