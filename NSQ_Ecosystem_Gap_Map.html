5. Problem-Point Register
Structured analysis per the architect-builder prompt format: Problem, Current State, Missing, Proposed Approach, Cost/Complexity, Decision.

5.1 ACI (Agentics Command Interface)

Problem: No functional command interface exists. The user operates through multiple disconnected Cowork session personas.
Current State: React tabletop mock. non-sequitur-web website pages (Sprint 1-3). 9 personas defined in SSOT.
Missing: Functional ACI shell. Persona routing engine. Command lane with execution. Morning Command Deck with live data. Live ops status board.
Proposed Approach: Build ACI as a standalone React application (separate from Hugo site). Start with persona selector + command lane + static Morning Brief. Wire to pipeline data for live status. Deploy on Cloudflare Pages at a subdomain (e.g., aci.justin-kuiper.com).
Cost / Complexity: MEDIUM. React + Tailwind + API stubs. 2-week MVP sprint. Higher complexity from persona context switching logic.
DECISION: BUILD NOW. ACI is the center of the ecosystem per SSOT. Nothing else works without it.

5.2 GCOL (Control Plane)

Problem: Tower coordination is entirely human-mediated. No automated routing, checkpoint enforcement, or operational visibility.
Current State: Tower model documented. Gunther repo (8 commits). Alistair Prime has socket IPC. Content pipeline has GitHub Actions.
Missing: Tower routing engine. State machine for GCOL flow. Checkpoint framework. Cross-observation mechanism. Telemetry/visibility layer.
Proposed Approach: Design GCOL as a lightweight state machine between ACI and execution layers. Use 6-step flow from mock as state model. Events emitted at each transition for ops status board.
Cost / Complexity: HIGH. State machine design, event system, multi-backend integration. 4-week sprint minimum.
DECISION: DESIGN NOW, BUILD IN PHASE 3. Get the state machine architecture right before implementing.

5.3 Skipjack Protocol Enforcement

Problem: Skipjack is doctrine but has zero runtime enforcement anywhere in the portfolio.
Current State: Fully documented in SSOT. Referenced in mock. Not implemented.
Missing: Validation middleware. Reconciliation checkpoints. Uncertainty signaling. Stale-state detection.
Proposed Approach: Implement as cross-cutting middleware. Start with content pipeline as proving ground. Require validation at every GitHub Actions workflow step. Extend to ACI and GCOL.
Cost / Complexity: MEDIUM. Pattern is straightforward (validate-or-halt). Challenge is consistent application.
DECISION: BUILD IN PHASE 2. Governance hard rule. Must be enforced before GCOL goes live.

5.4 Code Lab

Problem: Content Lab has architecture and a live pipeline. Code Lab has nothing.
Current State: Zero. Referenced in mock and SSOT but no architecture decisions, no design, no code.
Missing: Everything. Intake model. Pipeline stages. Convergence with Content Lab. RoboStack integration.
Proposed Approach: Design as parallel intake lane converging with Content Lab at backbone. Same stage-as-directory model. RoboStack provides execution. GitHub Actions provides workflow engine.
Cost / Complexity: HIGH. New subsystem from scratch. Must align with Content Lab patterns.
DECISION: DESIGN IN PHASE 3. Build after backbone convergence architecture defined.

5.5 RoboStack Integration

Problem: RoboStack is real infrastructure but completely disconnected from ACI.
Current State: Standalone. 45 commits, K3s, monitoring, CI/CD, MITRE threat model.
Missing: ACI-to-RoboStack bridge. Command execution pathway. Status reporting to ops board.
Proposed Approach: Thin API bridge. ACI sends requests through GCOL routing. RoboStack reports back via webhook. Start simple, evolve to full orchestration.
Cost / Complexity: MEDIUM. RoboStack already works. The bridge is the new work.
DECISION: BUILD IN PHASE 3. After ACI and GCOL foundations exist.

Phased approach. Each phase depends on the prior. Parallel work within phases encouraged.

Weeks 1–2  |  MEDIUM
Convert tabletop mock to functional React app. Persona selector with context switching. Morning Command Deck with static data. Command lane (text input → action stub). Status chips with live pipeline data. Deploy as standalone surface (separate from Hugo site).

Phase 2: Governance Layer

Weeks 3–4  |  MEDIUM

Phase 3: Control Plane (GCOL)

Weeks 5–8  |  HIGH

Phase 4: Integration + Polish

Weeks 9–12  |  MEDIUM

Phase 5: Hardening

Weeks 13+  |  MEDIUM
Skipjack full enforcement across all systems. Security review (Overwatch). Performance optimization. Monitoring and alerting. Documentation update. Terminal interface (Alistair Prime CLI).
