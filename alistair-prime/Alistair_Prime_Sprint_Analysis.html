2. Design Constraints from Architect Brief

Red Lines (Non-Negotiable)
  • Do not turn Prime into a chatbot. Prime executes structured tasks — it does not have conversations.
  • Do not collapse reasoning and execution. Alistair thinks. Prime acts. The separation is architectural, not cosmetic.
  • Do not overbuild. Modular, layered, runtime-oriented. Solo-operator scale.
  • NLP must not bypass orchestration. NLP sits between raw input and routing — it feeds into orchestration, never around it.

Integration Target: Robo Stack

Alistair Prime's next integration target is Robo Stack — the shared AWS + K3s infrastructure that hosts the Agentics platform. Prime will be the Cognitive Execution Core within Robo Stack. This means Prime must be containerizable, API-addressable, and stateless enough to run on K3s while maintaining SQLite persistence (or an equivalent).

Robo Stack Dependency Risk: Per the Gap Register (INF-02), K3s manifests exist (64 files) but none are applied to a running cluster. K3s must be operational before Prime can be deployed to Robo Stack. This is a Sprint 5 deliverable that blocks Sprint 6 tower activation. Plan Prime's sprint work to be infrastructure-independent — containerization and deployment specs can be designed now but deployment is blocked until K3s is live.
